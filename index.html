AIエージェント時代のデザイン再現手法

デザインは
DESIGN.md で決まる。

1ファイルにデザインルールを定義する。
AIエージェントがそれを読み、一貫性のあるUIを自動生成する。
人間が判断し、AIが再現する — 新しいデザインワークフロー。

50のテンプレートを見る DESIGN.md とは

AIにデザインを頼むたびに、
毎回違うものが出てくる。
色もフォントもレイアウトもバラバラ。

AIエージェントは優秀なビルダーだが、デザインの「正解」を知らない。 プロンプトのたびに異なる判断をする。フォントが変わる。色がブレる。余白が崩れる。 必要なのは、AIに渡す「デザインの仕様書」だった。

Markdownファイル
1つで、デザインが
統一される。

DESIGN.md は、プロジェクトのデザインルールを定義するMarkdownファイル。 カラー、フォント、余白、コンポーネントスタイル — すべての判断基準を1ファイルにまとめる。

プロジェクトルートに置くだけで、Claude Code をはじめとするAIエージェントが自動で読み込み、 定義に準拠したコードを生成する。

# DESIGN.md

## Color Palette & Roles
Primary: #0071E3
Background: #FFFFFF
Text: #1D1D1F
Surface: #F5F5F7

## Typography Rules
Font: "Inter", sans-serif
Base size: 16px
H1: Bold 48px

## Component Stylings
Button radius: 12px
Card radius: 16px
Nav height: 64px

How it works

3ステップで、デザインが変わる。

01

テンプレートを選ぶ

50種類のギャラリーから、プロジェクトに合うスタイルを選ぶ。 Stripe風のクリーンなFinTech、ブルータリズムの大胆さ、 禅の静寂 — どんな世界観も1クリックで。

02

プロジェクトルートに置く

ダウンロードした DESIGN.md を、プロジェクトのルートディレクトリに配置する。 必要に応じて値をカスタマイズ。 Claude Code は起動時にこのファイルを自動で読み込む。

your-project/
  DESIGN.md <-- ここに置く
  src/
  package.json
  ...
03

AIに「作って」と言う

あとは自然言語で指示するだけ。AIエージェントがDESIGN.mdを参照し、 カラー・フォント・余白・コンポーネントすべてが統一されたコードを生成する。 何度頼んでも、同じデザインが再現される。

$ claude
> LPを作成して

// DESIGN.md を自動読み込み
// カラー・フォント・余白が統一された
// HTML/CSS を生成

Why DESIGN.md

AIエージェント時代に
デザインが壊れる理由。

01

AIはデザインの
文脈を持たない

LLMは毎回ゼロからデザイン判断をする。前回のプロンプトで使った色やフォントを覚えていない。DESIGN.md が永続的な文脈になる。

02

Figmaのデザインは
コードに届かない

美しいFigmaファイルがあっても、AIエージェントはそれを読めない。Markdownなら、どのAIツールでも即座にパースできる。

03

デザイナー不在でも
一貫性を保てる

エンジニアだけのチーム、個人開発、スタートアップ。DESIGN.md があれば、デザインの専門知識がなくてもプロ品質のUIが生まれる。
