はじめ方

3ステップで、
はじめよう。

DESIGN.md をプロジェクトに置くだけ。
デザインの一貫性が、すぐに手に入る。

01

DESIGN.md をコピーする

テンプレートをダウンロードして、プロジェクトのルートに配置する。
Claude Code が自動で読み込み、デザインシステムに準拠したコードを生成する。

Terminal 
# プロジェクトのルートにコピー
cp DESIGN_md_template.md your-project/DESIGN.md

# 値を自分のプロジェクト用に書き換える
vi your-project/DESIGN.md
02

カラーとフォントを設定する

DESIGN.md の Color Palette と Typography Rules を、
自分のブランドに合わせて書き換える。わからない場合はデフォルト値をそのまま使える。

DESIGN.md 
## Color Palette & Roles

Primary:      #0071E3   （メインCTA・リンク）
Secondary:    #AF52DE   （サブアクション・タグ）
Background:   #FFFFFF   （ページ背景）
Surface:      #F5F5F7   （カード・セクション背景）
Text:         #1D1D1F   （本文テキスト）
Text-muted:   #86868B   （補足テキスト）
Accent:       #FF9500   （強調・警告）
Border:       #E8E8ED   （区切り線）
03

Claude Code でビルドする

DESIGN.md をプロジェクトルートに置いた状態で Claude Code を起動。
デザインシステムに準拠したHTML/CSS/コンポーネントが自動生成される。

Terminal 
# Claude Code を起動
claude

# デザインシステムに基づいてLPを生成
> DESIGN.md を使ってLPを作成して

# コンポーネントを個別に生成
> DESIGN.md に基づいてヘッダーコンポーネントを作って

FAQ

よくある質問

DESIGN.md とは何ですか?

プロジェクトのデザインルールを定義するMarkdownファイルです。カラー、フォント、余白、コンポーネントスタイルなど、デザインに関するすべての判断基準を1ファイルにまとめます。

デザインの知識がなくても使えますか?

はい。テンプレートにはデフォルト値が用意されており、そのまま使うだけで一貫性のあるデザインが実現できます。カスタマイズは後からいつでも可能です。

どのフレームワークで使えますか?

DESIGN.md はフレームワークに依存しません。HTML/CSS、React、Next.js、Vue、Svelte など、あらゆるプロジェクトで使用できます。

デフォルト値をそのまま使ってもいいですか?

はい。テンプレートのデフォルト値はそのまま使っても一貫性のあるデザインが実現できます。ギャラリーから好きなスタイルを選んで出発点にすることもできます。
